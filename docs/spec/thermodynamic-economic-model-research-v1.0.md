## 1. 调研范围与隐喻边界

linkrsp 将劳务与积分核算类比为**热力学过程**，用于：

- **产品叙事**：强调「无物理与核验证据则无产出」的可审计性；  
- **伦理对齐**：与 HDGP 管理域下「尊严底价 / 反诱导」叙事衔接（见可行性评估 §2–§3）；  
- **系统设计**：为 Clip、Harness、阻尼等机制提供**可沟通的直觉模型**。

**边界声明（与算法 spec 一致）**：下文**不**主张在物理学意义上证明「数字经济严格服从经典热力学能量守恒」；所谓「守恒」指 **核算口径上**「可观测投入—可记账产出」的**可审计绑定**，属**隐喻与工程约束**，非自然科学命题。

---

## 2. 范式转移：从信用叙事到可审计物理—信息成本

传统价值度量常依赖中心化信用或情绪波动。linkrsp 将 **LRS Credits** 叙事定位为：**与物理工时与核验强度绑定的权益凭证**（非货币、非证券承诺），其可辩护性来自证据链而非叙事强度（见可行性评估 §3.4、管理制度草案「不提供托管」）。

这一转移的**经济含义**是：把「积分增量」从**任意算法增发**，约束为**须对应可观测的 T/V 与经 Clip 约束的价值对价项**——从而在叙事上抑制无证据通胀，并与「影子账户」制度配套。

---

## 3. 「熵减 / 负熵」与 LRS-1.0 的对应关系

白皮书使用「生存熵减」隐喻（S 级任务作为物理世界中的负熵输入）。在本调研稿中，其与公式的对齐关系为：

| 隐喻表述 | 在 LRS-1.0 中的落点（详见算法 spec 第二章） |
|----------|---------------------------------------------|
| 时间的不可逆消耗 | $T_{\text{phy}}$（分钟） |
| 在场与核验成本 | $V_{\text{bit}}$ 与 V-Decay |
| 社会—经济对价与伦理约束 | $\operatorname{Clip}\bigl((D_{\text{base}}+\sum W_{\text{risk}})/K_{\text{global}},\,0.8,\,3.0\bigr)$ |
| 全网尺度上的「温度/难度」调节 | $K_{\text{global}}$ 与创世期锁定 |

**说明**：社区层面 **D 值偏离全网中位数** 等治理语义，与 **System Damping**、**R-008** 等规则一致（可行性评估 §3.3、§6），算法 spec 第三章亦有摘要。

## 4. 物理存证与「作弊成本物理化」（深度调研摘要）

### 4.1 近场核验的经济逻辑

**V=2** 路径将可信边界从「云端脚本」推向「在场对碰」，使套利者的边际成本包含**位移、时间与协调**（可行性评估 §8.1）。调研结论与产品策略一致：**不以绝对防伪造为唯一目标**，而以 **V-Decay + Harness + 审计追溯** 提高系统性作弊成本至不经济区间。

### 4.2 与欺诈经济学的接口

当作弊的期望成本（含设备、人力、处罚概率）超过积分的社会对价期望时，理性攻击面收敛——该判断依赖参数化与治理数据，**须由上线后实证迭代**，本文档不固化具体数值。

---

## 5. 长期演化：审计周期与权力结构

**90 天归一化审计**、**治理权与积分的非线性挂钩**，在叙事上对应「防止金元政治」与「系统自净化」（可行性评估 §3.4）。技术实现上，AI-Oracle / LLM 仅处理**语义灰区**；物理矛盾优先走 **R 类规则**（可行性评估 §5.3、§6）。**HDAG 等证据编排模型**若采用，属工程可选方案，见 [`../engineering/technology-strategy-v1.0.md`](../engineering/technology-strategy-v1.0.md) 的「可替换组件」原则。

---

## 6. 红队视角（与可行性评估 §8 对齐）

本调研不重复 §8 条文，仅强调：**物理锚失灵、K_global 滞后、自治定价偏差、LLM 成本、申诉链缺失** 五类风险与缓解，以可行性评估为准；极端场景下的全局锁定与人工接管须与合规与数据最小化一并论证。

---

## 7. 结语

热力学经济模型调研的价值在于：为 **LRS-1.0** 提供**可对外解释**的连续叙事，并把「隐喻」与「可执行约束」分离——**执行约束以算法 spec + 可行性评估为 canonical**。
