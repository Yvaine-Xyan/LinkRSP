## 2. 「公开记录页面」指什么

此处**不要求**独立 Web 应用。最小可审计做法是：

1. **本仓库内的只追加 Markdown 日志**：[`../operations/appeal-log.md`](../operations/appeal-log.md)  
   - 每条记录经 **Pull Request** 合并后，天然具备 **Git 提交哈希、作者、时间戳**，便于第三方核对「是否回应过某类申诉」。  
2. **可选增强**：GitHub **Issues / Discussions** 使用固定标签（如 `appeal`），与上述日志二选一或并存；仍以「可链接、可引用」为准。

**不是**：必须与产品首页同域的动态页面；任何能**稳定公开引用**且**防静默删除**（在 Git 托管商政策范围内）的载体均可视为满足「公开记录」的工程等价物。
