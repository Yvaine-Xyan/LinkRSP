# 版本策略（发布物 vs 文字稿）

> 维护者约定：在**系统正式达到上线标准**之前，或维护者**未书面指定**新的发布版本号之前，**发布层面**的版本标识**一律不得高于 v1.0**。

## 适用范围（「不得高于 v1.0」约束的是谁）

| 类别 | 是否受「≤ v1.0」约束 | 说明 |
|------|----------------------|------|
| **文字文档**（Markdown、白皮书、草案、评述） | **不适用本条的数值封顶** | 文稿可使用 **v1.0 Alpha**、以及文内对**未来规格迭代**的表述（例如「算法规格说明书下一迭代」）；此类用语**不代表**已对外发布更高 semver。 |
| **发布物与工程标识** | **适用** | 含：Git **tag**、**API 路径版本**（如 `/v1/`）、包管理器 **semver**（npm/cargo 等）、可执行组件 **对外宣称版本**、对外下载文件名版本。上述在未达上线标准或未指定前，**不得**出现 **v1.1、v2.0** 等高于 **v1.0.x** 的发布语义。 |
| **仓库内工程草案文件名** | 建议 ≤ v1.0 语义 | 例如 `architecture-v0.1` 表示**草案层级**，与发布版本区分。 |

## 生效与修订

- 维护者指定「首个对外发布版本」或「上线标准达成」后，可修订本策略并同步 `README`。
- 文档索引：[`docs/README.md`](../README.md)。
